# Streamlit reference for mart-forge agents

A condensed reference an agent can read once instead of fetching docs
mid-session. For full docs see https://docs.streamlit.io.

## Minimal app

```python
import streamlit as st

st.set_page_config(page_title="my mart", layout="wide")
st.title("My Mart Dashboard")
st.write("Hello from Streamlit.")
```

Run it:
```
streamlit run app.py --server.port 8503
```

## Data loading and caching

```python
import duckdb
import pandas as pd

@st.cache_resource
def get_conn():
    return duckdb.connect("warehouse.duckdb")

@st.cache_data(ttl=600)
def load_metrics(date: str) -> pd.DataFrame:
    return get_conn().execute(
        "SELECT * FROM gme_ads_market_dashboard WHERE pull_date = ?",
        [date]
    ).df()
```

`@st.cache_resource` caches a connection across reruns.
`@st.cache_data` caches DataFrames; use a TTL so freshness matters.

## Status badges (used for link-status display)

```python
STATUS_BADGE = {
    "verified":     "✅",
    "proxy":        "⚠️ proxy",
    "stale":        "⏳ stale",
    "unsupported":  "🚫 unsupported",
    "pending":      "⏱ pending",
}

def metric_with_badge(label: str, value: str, status: str):
    badge = STATUS_BADGE.get(status, "?")
    st.metric(label=f"{label} {badge}", value=value)
```

Per `G-HONEST-LABEL` (SPEC §9), every dashboard metric MUST render the
badge. Removing the badge call is a CI failure.

## Health check endpoint

Streamlit exposes `/_stcore/health` returning `ok` when the app is
running. CI uses this for the dashboard smoke test:

```bash
curl -fsS http://localhost:8503/_stcore/health
```

## Common pitfalls

- **Reruns on every input.** Streamlit re-runs the whole script on
  every widget interaction. Wrap expensive work in `@st.cache_data`
  or `@st.cache_resource`.
- **Hard-coded data paths.** Use an env var for the warehouse path so
  CI and local share the same code.
- **No global state across users.** Streamlit runs per-session;
  shared state needs an external store (Redis, DB).
- **Connection per session.** Avoid creating one DuckDB connection
  per widget interaction — cache it.
